# Codeown UI Design System (v1.0)

## Overview

This document defines the official UI/UX design system for **Codeown**.
The goal is to maintain a consistent, minimal, and developer-focused interface across all pages and components.

**Core Philosophy:**

> Minimal. Functional. Developer-first.

The design must avoid generic or “AI-generated” aesthetics and instead feel intentional, clean, and practical.

---

## 1. Design Principles

### 1.1 Minimalism Over Decoration

* No unnecessary shadows, gradients, or visual noise
* Use spacing and typography instead of heavy UI elements

### 1.2 Function First

* Every UI element must serve a clear purpose
* Avoid purely decorative components

### 1.3 Consistency is Mandatory

* Same spacing, typography, and structure across all pages
* Reusable components must look and behave identically

### 1.4 Developer-Oriented UX

* Fast navigation
* Clear hierarchy
* Information-first layout (similar to dev tools)

---

## 3. Typography

### Font:

* Primary: **Plus Jakarta Sans**

### Font Weights:

* Regular: 400
* Medium: 500
* SemiBold: 600

### Font Sizes:

* Page Title: 28–32px
* Section Title: 18–20px
* Body Text: 14–16px
* Small Text: 12–13px

### Rules:

* No more than 3 font sizes per section
* Use font weight for hierarchy, not color

---

## 4. Spacing System

### Base Unit:

* 8px grid system

### Common Spacing:

* XS: 4px
* SM: 8px
* MD: 16px
* LG: 24px
* XL: 32px

### Rules:

* Always use consistent spacing
* Never use random values

---

## 6. Component Design Rules

### 6.1 No Card Overuse

Avoid heavy “card-based” layouts.

Instead:

* Use spacing
* Use dividers (thin lines)
* Use vertical stacking

---

### 6.2 Borders

* Use subtle borders: `1px solid #1a1a1a`
* Avoid shadows

---

### 6.3 Buttons

#### Primary Button:

* Background: white
* Text: black

#### Secondary Button:

* Background: transparent
* Border: 1px solid

#### Rules:

* No rounded exaggerated corners
* Border-radius: 6–10px max

---

### 6.4 Inputs

* Dark background
* Subtle border
* No glow effects

---

## 11. Interaction Design

### Required:

* Subtle hover effects
* Fast transitions (150–250ms)
* No exaggerated animations

### Avoid:

* Bounce effects
* Over-animation

---

## 12. UX Rules

### 12.1 Speed First

* Interface must feel instant

### 12.2 Clarity Over Creativity

* User should never feel confused

### 12.3 Reduce Friction

* Fewer clicks = better

---

## 13. Anti-Patterns (STRICTLY FORBIDDEN)

* ❌ Gradient backgrounds
* ❌ Bright colors
* ❌ Overuse of cards
* ❌ Heavy shadows
* ❌ Random spacing
* ❌ Inconsistent typography
* ❌ AI-generated generic layouts

---

## 14. Final Guideline

> If a design looks “cool” but doesn’t improve usability → remove it.

> If a design is simple but effective → keep it.

---

## End Goal

Codeown should feel like:

* A professional developer tool
* Clean and distraction-free
* Built by developers, for developers
